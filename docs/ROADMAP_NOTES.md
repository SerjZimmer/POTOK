# Дорожная карта: Заметки

Документ описывает текущее состояние и план развития модуля «Заметки» (Notes) с учётом перехода к онлайн‑режиму и последующего масштабирования.

## 0. Текущее состояние

- Клиент: Flutter, список папок и заметок; CRUD заметок; свайп‑удаление; счётчики по папкам.
- Сервер: Go, Gorilla Mux (переход на chi планируется), SQLite (WAL, busy_timeout). Эндпоинты `/folders`, `/notes`, `/folders/{id}/notes` и пр.
- БД: файл `backend/potok.db`.

Ограничения: нет авторизации, нет поиска/индексации, нет синхронизации и версионирования.

---

## 1. Улучшения UX/функционала (офлайн/онлайн‑готовность)

- Редактор заметок: Markdown подсветка (минимум), предпросмотр.
- Поиск: локальный FTS5 по `title/content` (мобильные/desktop). Для Web — компактный индекс в памяти.
- Масштабируемая структура папок: вложенные папки (опционально), сортировки.
- Избранное/пины, быстрые фильтры (только в текущей папке / по всем).
- Undo/Redo на клиенте для быстрых правок.

---

## 2. Сервер (онлайн)

- Переход на PostgreSQL
  - Схема: `folders`, `notes`, `users`, `labels` (опционально), FTS.
  - Индексы: `notes(folder_id, updated_at)`, FTS по `title/content`.
- API (OpenAPI): CRUD, листинги с пагинацией (cursor‑based), фильтры, сортировки.
- Версионирование/конкурентный доступ
  - `ETag` на заметку, `If-Match` при PUT.
  - История изменений (audit trail) — для «сравнить версии».
- Синхронизация
  - Модель, аналогичная календарю: delta + anchors, idempotency keys.
  - Конфликты: Last‑Writer‑Wins, опционально merge текстов (в будущем CRDT/OT для совместного редактирования).
- Поиск серверный: PostgreSQL FTS (или Meilisearch/Elastic при росте).

---

## 3. Масштабирование (до 200 000 пользователей)

- API: stateless узлы за LB, кэширование часто читаемых заметок в Redis (LRU).
- БД: PG‑кластер, реплики для чтения, шардинг по пользователям (при необходимости).
- Наблюдаемость: Prometheus, Grafana, логгирование (JSON), трассировка (OTel).
- Безопасность: JWT/OAuth2, TLS, WAF/Rate‑limits, защита от утечек (изоляция по tenant_id).

---

## 4. Миграция данных и совместимость

- Из SQLite → PostgreSQL: миграция через временный экспорт/импорт (CSV/SQL) либо ETL‑задачу в Go.
- Совместимость API: сохранить текущий contract для фронта, добавить версионирование `/v1`.

---

## 5. План релизов

1) Поиск (FTS5) + UX правки (Markdown/preview, сортировки, пины).
2) OpenAPI, chi, ETag, пагинация, базовая авторизация.
3) PostgreSQL + миграция данных, серверный поиск.
4) Синхронизация delta/anchors, offline‑мёрджи, конфликты.
5) Масштабирование и наблюдаемость.
