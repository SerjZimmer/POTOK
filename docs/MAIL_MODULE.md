# Модуль "Почта" для POTOK

## Обзор

Модуль "Почта" предоставляет полнофункциональную систему управления электронной почтой с поддержкой множественных аккаунтов, различных провайдеров и офлайн-первого UX.

## Архитектура

### Backend (Go)
- **Модели**: `model/types.go` - все сущности (Account, Mailbox, Thread, Message, Attachment)
- **Провайдеры**: `provider/` - абстракция для различных почтовых сервисов
- **Сервисы**: `service/` - бизнес-логика (AccountService, ThreadService, SendService)
- **HTTP**: `http/router.go` - REST API с использованием Chi router

### Frontend (Flutter)
- **Экраны**: `presentation/` - UI для аккаунтов, тредов, просмотра писем, compose
- **Данные**: `data/` - Drift схемы, репозитории, провайдеры
- **Домен**: `domain/` - бизнес-логика и сущности

## API Endpoints

### Аккаунты
- `GET /v1/mail/accounts` - список аккаунтов
- `POST /v1/mail/accounts` - создание аккаунта
- `GET /v1/mail/accounts/{uid}` - получение аккаунта
- `PATCH /v1/mail/accounts/{uid}` - обновление аккаунта
- `DELETE /v1/mail/accounts/{uid}` - удаление аккаунта

### Ящики
- `GET /v1/mail/accounts/{uid}/mailboxes` - список ящиков аккаунта

### Треды
- `GET /v1/mail/accounts/{uid}/threads` - список тредов
- `GET /v1/mail/threads/{id}` - получение треда с сообщениями

### Сообщения
- `GET /v1/mail/messages/{id}` - получение сообщения
- `POST /v1/mail/messages/{id}/flags` - установка флагов
- `POST /v1/mail/messages/{id}/move` - перемещение сообщения
- `DELETE /v1/mail/messages/{id}` - удаление в корзину
- `DELETE /v1/mail/messages/{id}/permanent` - окончательное удаление

### Отправка
- `POST /v1/mail/send` - отправка сообщения
- `POST /v1/mail/drafts` - создание/обновление черновика

### OAuth
- `POST /v1/mail/oauth/{provider}/start` - начало OAuth
- `POST /v1/mail/oauth/{provider}/callback` - завершение OAuth

### Вложения
- `GET /v1/mail/messages/{id}/attachments/{aid}` - скачивание вложения

## База данных (Drift)

### Основные таблицы
- `accounts` - почтовые аккаунты
- `mailboxes` - ящики/папки
- `threads` - треды сообщений
- `messages` - сообщения
- `attachments` - вложения
- `outbox_queue` - очередь исходящих
- `contacts` - контакты для автокомплита
- `tokens` - зашифрованные токены

### Индексы
- FTS5 для поиска по теме, адресам, телу
- Составные индексы для производительности
- Индексы по времени для синхронизации

## Безопасность

### Хранение секретов
- **Flutter**: Keystore/Keychain через secure storage
- **Backend**: Шифрование токенов (KMS/переменные среды)
- **Логи**: Маскирование PII, редактирование личных данных

### TLS
- Обязательное использование TLS для всех соединений
- Проверка сертификатов
- Поддержка STARTTLS

### Санитация
- HTML-санитация (удаление скриптов, опасных атрибутов)
- Блокировка удаленных трекеров
- Валидация вложений

## Синхронизация

### Стратегия
- **Офлайн-первый**: локальный кеш + фоновый sync
- **Инкрементальная**: загрузка только изменений
- **Конфликты**: last-writer-wins с провайдер-спецификой

### Фоновые процессы
- Периодический pull (настраиваемый интервал)
- Long-poll/вебхуки (где поддерживается)
- Экспоненциальные ретраи для неудачных попыток

## Развертывание

### Переменные окружения
```bash
# OAuth провайдеры
GMAIL_CLIENT_ID=your_gmail_client_id
GMAIL_CLIENT_SECRET=your_gmail_client_secret
OUTLOOK_CLIENT_ID=your_outlook_client_id
OUTLOOK_CLIENT_SECRET=your_outlook_client_secret

# Шифрование
ENCRYPTION_KEY=your_encryption_key

# База данных
DATABASE_URL=file:./potok_mail.db
```

### Запуск
```bash
# Основной сервер (включает модуль "Почта")
make run-backend

# Отдельный сервер модуля "Почта"
cd backend/cmd/mail-server
go run main.go
```

## Устранение неполадок

### Частые проблемы
1. **Ошибки OAuth**: проверьте client_id/secret, redirect_uri
2. **IMAP соединения**: проверьте порты, TLS настройки
3. **Синхронизация**: проверьте логи, сетевые соединения
4. **Производительность**: мониторьте размер БД, индексы

### Логи
- Backend: `slog` с уровнем INFO
- Frontend: `BoardsLogger` для отладки
- База данных: SQLite WAL логи
